# 战役二完成报告：全自动调度中心

## 📊 实现对比

### 初版实现（一次性执行）
```python
# autopilot_service.py (初版)
async def start_autopilot(novel_id, target_acts=3):
    session = AutopilotSession(novel_id, target_acts)
    await self._run_state_machine()  # 跑完就停
    return session
```

**问题：**
- ❌ 不是守护进程，执行完就停止
- ❌ 状态不持久化，进程重启丢失
- ❌ 缺少缓冲章策略
- ❌ Novel 模型缺少状态字段

---

### 守护进程架构（最终版）
```python
# autopilot_daemon.py (最终版)
def run_forever(self):
    while True:
        active_novels = self._get_active_novels()  # 从数据库捞
        for novel in active_novels:
            await self._process_novel(novel)  # 状态机调度
        time.sleep(5)  # 轮询间隔
```

**优势：**
- ✅ 真正的守护进程，死循环轮询
- ✅ 状态持久化到数据库，可恢复
- ✅ 集成缓冲章策略（ACT_PLANNING）
- ✅ 集成节拍放大器（WRITING）

---

## 🎯 两大战术植入点

### 1️⃣ 缓冲章策略（ACT_PLANNING 阶段）

**问题诊断：**
- 原型生成的 30 章节奏过载，300 章的故事压缩到 30 章
- 缺少"日常水文"、"装逼打脸"的缓冲期

**解决方案：**
```python
# autopilot_daemon.py: _handle_act_planning()
needs_buffer = self._needs_buffer_chapters(novel)

if needs_buffer:
    # 强制插入 2 章缓冲章
    # 第1章：战后休整/疗伤/盘点收获（日常向）
    # 第2章：装逼打脸/路人震惊/展示战利品（爽点密集）
```

**效果：**
- 每经历 1 场大战，强制插入 1-2 章缓冲
- 读者有喘息空间，不会看得太累
- 爽点密度更合理

---

### 2️⃣ 节拍放大器（WRITING 阶段）

**问题诊断：**
- AI 在单章内塞入太多动作节拍
- 情绪铺垫缺失，"该虐 5 章的地方 1 章和好"
- 战斗场面干瘪，缺感官刺激

**解决方案：**
```python
# context_builder.py: magnify_outline_to_beats()
if "争吵" in outline:
    beats = [
        Beat("场景氛围描写：压抑的环境、紧张的气氛", 500, "sensory"),
        Beat("冲突爆发：主角的质问、对方的反应", 800, "dialogue"),
        Beat("情绪细节：内心独白、回忆闪回", 700, "emotion"),
        Beat("冲突结果：决裂、离开（不要轻易和好）", 500, "action"),
    ]
```

**效果：**
- 单章大纲拆分为 4 个微观节拍
- 每个节拍 500-800 字，强制 AI 放慢节奏
- 增加感官细节、情绪铺垫、战斗描写

## 🚀 使用方法

### 1. 启动守护进程
```bash
python scripts/start_daemon.py
```

### 2. 在数据库中标记小说为 RUNNING
```python
novel = novel_repository.get_by_id(NovelId("test_001"))
novel.autopilot_status = AutopilotStatus.RUNNING
novel.stage = NovelStage.MACRO_PLANNING
novel_repository.save(novel)
```

### 3. 守护进程自动接管
- 每 5 秒轮询一次数据库
- 自动执行状态机：MACRO_PLANNING → ACT_PLANNING → WRITING → AUDITING
- 状态持久化，进程重启可恢复
